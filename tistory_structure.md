# 티스토리 관리자 구조 파악 — 웹사이트 편법 관점

> 작성일: 2026-05-20 KST
> 대상 블로그: baekhahn.tistory.com (현재 스킨: Odyssey)
> 목적: "블로그 엔진을 웹사이트처럼 쓰기" 위해 어떤 입력 필드가 어디로 출력되는지 정리.
> 출처: 티스토리 관리자(`/manage/*`) 화면 직접 탐색.

---

## 1. 콘텐츠 타입 (입력 → 출력)

티스토리에는 3가지 콘텐츠 타입이 있고, 각자 별도 URL 공간과 별도 관리 화면을 가짐.

### 1.1 글 (Post) — 가장 자유로움

**입력 화면**: `/manage/newpost/` (글쓰기), `/manage/posts` (목록)

| 입력 필드 | 출력 / 용도 |
|---|---|
| **카테고리** (드롭다운) | 카테고리 페이지에서 묶임. URL: `/category/카테고리명` |
| **제목** | 글 상세의 `[##_article_rep_title_##]`, 목록 카드 제목 |
| **본문** (3가지 모드) | 글 상세의 `[##_article_rep_desc_##]` 로 **HTML 그대로** 출력 |
| └ 기본모드 | TinyMCE WYSIWYG |
| └ 마크다운 | 마크다운 → HTML 변환 |
| └ **HTML** | **다크 코드 에디터, 임의 HTML 직접 입력** ← **웹사이트 편법의 핵심** |
| **태그** | `#태그` 형식, 태그 페이지에 묶임 |
| **공개여부** | 공개 / 공개(보호: 비번) / 비공개 |
| **댓글 허용** | on/off |
| **홈주제** | 티스토리 홈(메인 피드) 분류 — 외부 노출용, 사이트 운영에는 무관 |
| **발행일** | 예약 발행 가능 |
| **URL slug** | `https://baekhahn.tistory.com/entry/[자유 입력]` ← **임의 라우팅 가능** |
| **대표이미지** | 목록 카드 썸네일, OG 이미지 |
| **CCL 설정** | 라이선스 표기 |

**핵심**: HTML 모드 + URL slug 커스텀 = "임의 페이지" 를 글로 만들 수 있음.

### 1.2 페이지 (Page) — 카테고리·태그 없는 정적 페이지

**입력 화면**: `/manage/page?returnURL=/manage/pages`, `/manage/pages` (목록)

- 글쓰기와 동일한 에디터 (3가지 모드 동일).
- **차이점**: 카테고리·태그 없음. 상위 페이지 선택 가능 (계층).
- 용도: About / 회사소개 / Contact 같은 "글 흐름과 무관한 정적 페이지".

### 1.3 공지 (Notice) — 상단 고정용

**입력 화면**: `/manage/notice`

- 동일한 에디터, "공지사항" 라벨.
- 카테고리·태그 없음.
- 사이드바의 "공지사항" 모듈, 메인 페이지 상단에 노출되는 콘텐츠 타입.

### 1.4 서식 (Template) — 글 보일러플레이트

**입력 화면**: `/manage/template`

- 콘텐츠가 아니라 **글 작성 시 끼워넣는 템플릿**.
- 같은 레이아웃(예: 캠페인 페이지) 의 글을 여러 개 만들 때 베이스로 사용.

## 2. 분류 체계

### 2.1 카테고리 (`/manage/category`)

- 트리 구조 (대분류 → 소분류, 펼치기/접기).
- 최대 500개.
- 드래그 앤 드롭 정렬.
- 카테고리명 글자수 제한: 27자.
- 옵션: 글수 표시 / "새 글" 뱃지 (N일 이내).
- **URL 패턴**: `/category/카테고리명` 또는 `/category/대/소`.

### 2.2 태그

- 글 작성 시 자유롭게 추가.
- 별도 관리 화면 없음 (글에서 입력 = 태그 생성).
- 사이드바 "태그 클라우드" 모듈, 메뉴의 "태그 검색" 항목으로 노출.

## 3. 네비게이션 (GNB)

**입력 화면**: `/manage/design/menu`

기본 메뉴 3개: **홈 / 태그 / 방명록**. 드래그 정렬, 추가/삭제 자유.

**메뉴 추가 시 타입 5종**:

| 타입 | 입력값 | 결과 |
|---|---|---|
| 기본 메뉴 | (없음) | 홈/태그/방명록 중 선택 |
| **페이지** | 페이지 선택 | 1.2 의 페이지로 링크 |
| **카테고리** | 카테고리 선택 | 카테고리 페이지로 링크 |
| 태그 검색 | 태그명 | 해당 태그 페이지 |
| **직접 입력** | 임의 URL + 새창 옵션 | 외부 링크 |

**핵심**: 카테고리·페이지·외부URL 자유 조합 → 웹사이트형 GNB (About / Works / Journal / Contact 등) 구성 가능.

## 4. 사이드바 (`/manage/design/sidebar`)

두 슬롯: **사이드바1 / 사이드바2**. 모듈을 드래그로 배치.

### 4.1 기본 모듈 (드래그 후보)

- 프로필
- 카테고리 메뉴
- 태그 (클라우드)
- 최근글과 인기글
- 최근댓글
- 공지사항
- 페이스북 트위터 플러그인
- 글 보관함 (월별 아카이브)
- 달력 모듈
- 방문자수
- [플러그인] Daum 검색창

### 4.2 사용자 모듈

- 별도 영역 존재 — **임의 HTML 모듈** 추가 가능 (사이드바에 광고/CTA/배너 등 직접 작성).

### 4.3 사이드바 설정

- 최근 글: 표시 개수 / 글자수 (예: 3개 / 70자)
- 최근 댓글: 개수 / 글자수
- 최근 공지: 개수 / 글자수
- 글 보관함: 표시할 개월수
- 링크 모듈: 글자수 제한

## 5. 스킨 편집 (`/manage/design/skin/edit`)

### 5.1 우측 GUI 설정 (HTML 안 건드리고 가능한 것)

- **홈 설정**: 최신 글 모드 / 커버 모드 (탭 선택). 커버 모드는 "베이지 타입" 등 커버 아이템 추가 가능.
- **기본 설정**: 글 목록 글 수, 태그 클라우드 순서(인기순/이름순), 기본 목록 스타일(이미지 크기 고정 등).
- **타이틀 헤더**: 로고 이미지 (높이 18px 권장).
- **PC 레이아웃**: 사이드바 형태 (1단 서랍 사이드바 등).
- **목록 탐색**: PC = 페이지 나열, 모바일 = 더보기 버튼.
- **내비게이션**: 사용 안 함 등.

### 5.2 HTML 편집 (`#/source/html`)

**구조**: 단일 `skin.html` 에 모든 라우트(홈/글/카테고리/태그/방명록/공지)가 조건 블록으로 분기.

**치환 변수 패턴**: `[##_변수명_##]`
- 페이지 메타: `[##_page_title_##]`, `[##_title_##]`, `[##_body_id_##]`
- 글: `[##_article_rep_title_##]`, `[##_article_rep_desc_##]` (본문 HTML), `[##_article_rep_date_##]`, `[##_article_rep_thumbnail_url_##]`
- 블로그: `[##_blog_link_##]`, `[##_var_*_##]` (스킨 변수)
- 헤더 슬로건: `[##_var_headersloguntitle_##]`, `[##_var_headersloguntext_##]`
- 로고: `[##_var_logoImage_##]`

**조건 블록 패턴**: `<s_*>...</s_*>`
- `<s_t3>` — 메인 wrapper
- `<s_if_var_logoImage>` / `<s_not_var_logoImage>` — 변수 유무 분기
- `<s_if_var_headersloguntitle>` — 슬로건 표시 분기
- (추정) `<s_index>` 홈 / `<s_article_rep>` 글 상세 / `<s_category_rep>` 카테고리 / `<s_tag_label>` 태그 / `<s_notice_rep>` 공지 / `<s_guest>` 방명록

### 5.3 CSS 편집 (`#/source/css`)

- 일반 CSS 파일. reset/normalize 기반.
- 외부 폰트(jsdelivr 등) CDN 링크 자유롭게 추가 가능.

### 5.4 파일 업로드

- 이미지, 폰트, JS 등 스킨 리소스를 업로드. `images/파일명` 형식으로 참조.

## 7. 웹사이트 편법 적용 시나리오

티스토리를 "회사 홈페이지" 처럼 쓰려면 다음 매핑이 가능함.

| 웹사이트 페이지 | 티스토리 구현 |
|---|---|
| 홈 (랜딩) | **스킨 편집의 `<s_index>` 블록을 직접 작성** + 홈 설정에서 "커버 모드" 활용. 또는 첫 페이지를 특정 글로 리다이렉트. |
| About / 회사소개 | **페이지 1개** (`/pages/about` 류) + 본문 HTML 직접 작성 |
| Works / Portfolio | **카테고리 1개** ("Works") + 각 프로젝트 = 글 1개. 또는 단일 글 안에 그리드 HTML 박아넣기 |
| Journal / Blog | **카테고리** ("Journal") + 일반 글 운영 |
| Contact | **페이지** 1개 + HTML 폼 (외부 폼 서비스 임베드) |
| GNB | 메뉴 설정에서 페이지/카테고리/외부URL 조합 |
| 푸터 / 사이드바 광고 영역 | 사이드바 "사용자 모듈" 에 HTML 직접 |

### 적용에서 가장 강력한 카드 3개

1. **글 본문 HTML 모드** — 한 글 안에 임의 디자인을 풀로 박을 수 있음. 사실상 "글 = 페이지" 로 쓸 수 있음.
2. **스킨 HTML 직접 편집** — `<s_index>`, `<s_article_rep>` 등 라우트별 블록을 우리가 짠 디자인으로 통째 교체.
3. **메뉴 + 카테고리 + URL slug** — 라우팅을 우리 마음대로 설계.

### 제약사항

- URL 패턴은 티스토리가 정한 프리픽스(`/entry/`, `/category/`, `/tag/`, `/guestbook` 등) 안에서만 가능.
- 포스트 주소는 숫자형(`/123`) / 문자형(`/post-slug`) 둘 중 택1, 전역 적용.
- 페이지(`/pages`) 는 카테고리·태그가 없음 — "분류된 정적 페이지" 는 불가, 카테고리에 글로 넣어야 함.
- 서버사이드 동적 로직 불가 — 클라이언트 JS 와 티스토리 변수만으로 처리.

## 8. 다음 단계 제안

1. **`design.md`** (사용자가 줄 디자인 레퍼런스 URL 분석 결과) 와 **본 문서** 를 합쳐, "어떤 페이지를 어떤 티스토리 구조로 매핑할지" 페이지별 매핑 표 작성.
2. 현재 `index.html` 의 TEMPLATES (theme-base / theme-campaign / post) 가 어떤 티스토리 블록(`<s_index>` / `<s_cover_group>` / `<s_article_rep>`) 으로 치환될지 1:1 대응표 작성.
3. 치환 변수를 `[##_*_##]` 형식으로 HTML 에 삽입 (예: 더미 제목 "건축에 대한 깊은 사고" → `[##_article_rep_title_##]`).
4. `index.html` TEMPLATES → `skin.html` 빌드 스크립트에 변수 치환 단계 추가.
